# History 페이지 UI/UX 개편 스펙 (UI/UX 리드)

## 개요

- **목적**: `/history` 페이지를 전체 서비스 UI/UX와 동일한 디자인 시스템으로 통합하고, 사용자 플로우·접근성·일관성을 강화한다.
- **리드 역할**: UI/UX (사용자 플로우, 와이어프레임, 디자인 시스템)

## 요구사항

### 기능 유지
- 생성된 포스트 목록 표시 (페이지네이션, 검색, 정렬)
- 포스트 카드: 제목, 메타(날짜·글자 수), 키워드 배지, 원문 링크, 보기/삭제
- 모달: 전체 내용 보기, 복사, 다운로드
- 빈 상태 시 CTA로 메인 생성 페이지 유도

### 디자인 시스템 연동
- **스타일 시트**: `main.css`, `sanity_style.css`, `common.css` 사용 (index와 동일)
- **토큰**: `_variables.css` 기반 (색상, 간격, 반경, 타이포)
- **컴포넌트**: `card-custom`, `section-title`, `btn-custom`, `btn-outline-custom`, `form-control-custom`, `badge-custom`
- **레이아웃**: `container-custom`, 상단 네비게이션 높이에 맞춘 패딩

### 페이지 구조
1. **페이지 헤더**: 메인과 톤 맞는 "생성 기록" 타이틀 + 한 줄 설명 + 연결 서비스 링크(GAEO, 프롬프트 메이커)
2. **검색·필터**: 카드 형태로 감싸기, 검색 입력·정렬 셀렉트 디자인 시스템 적용
3. **포스트 목록**: 카드 리스트, 호버/포커스 접근성
4. **빈 상태**: 아이콘 + 문구 + "포스트 생성하기" 버튼
5. **푸터**: 고정 푸터 유지, 저작자·연결 서비스 링크로 서비스 일관성
