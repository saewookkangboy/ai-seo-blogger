# 전체 서비스 반응형·Seamless UI/UX 스펙 (UI/UX 리드)

## 개요

- **목적**: 웹과 모바일에서 동일한 디자인 시스템으로 seamless하게 보이도록 반응형을 일원화하고, 터치·가독성·접근성을 강화한다.
- **리드 역할**: UI/UX (사용자 플로우, 디자인 시스템, 반응형)

## 요구사항

### 1. 브레이크포인트·전역
- **단일 브레이크포인트**: `768px` (Bootstrap `md`) — 모바일(<768px), 데스크톱(≥768px)
- **뷰포트**: `width=device-width, initial-scale=1` (필요 시 `viewport-fit=cover`로 노치 대응)
- **body**: `overflow-x: hidden`, `min-height: 100vh`로 가로 스크롤 방지·최소 높이 보장
- **safe area**: 노치 기기에서 하단·좌우 패딩 적용 가능하도록 변수 또는 유틸 클래스

### 2. 터치·입력
- **터치 타겟**: 버튼·링크 최소 44×44px (또는 min-height 44px)
- **입력 폰트**: 모바일에서 `input`/`select`/`textarea` `font-size: 16px` 이상 — iOS 줌 방지
- **포커스**: `focus-visible` 아웃라인 유지로 키보드 접근성

### 3. 네비게이션
- **모바일**: 햄버거 → collapse 메뉴, 메뉴 링크·버튼 터치 영역 확보
- **모바일 메뉴 항목**: 생성하기, 기록, 타겟 분석, 관리자, GAEO, 프롬프트, **로그인/로그아웃** 동일 노출
- **sticky**: 상단 고정, z-index 일관

### 4. 페이지별 반응형
- **메인(/)**: 히어로 제목·부제·연결 서비스 링크 줄바꿈·폰트 스케일; 탭·폼·결과 카드 단일 컬럼 또는 여백 조정; 결과 영역 버튼 그룹 wrap
- **기록(/history)**: 검색 행 모바일에서 세로 스택; 푸터 링크 wrap; 카드·페이지네이션 터치 친화
- **로그인/관리자**: 폼·버튼 전체 너비 또는 적절한 max-width

### 5. 컴포넌트
- **카드**: 모바일 패딩 축소, 제목·본문 줄바꿈
- **결과 영역**: 헤더(제목+배지+버튼) flex-wrap; 복사/다운로드 버튼 작은 화면에서 세로 정렬 또는 wrap
- **모달**: 소형 화면에서 fullscreen 또는 max-height + 스크롤
- **테이블**: 가로 스크롤 또는 카드 형태 전환(기존 table-responsive 유지)

### 검증 기준
- [ ] 320px~428px(모바일), 768px(태블릿), 1024px(데스크톱)에서 레이아웃·가독성 확인
- [ ] 모든 인터랙티브 요소 터치 영역 ≥44px
- [ ] 모바일에서 input/select 포커스 시 브라우저 자동 줌 없음(16px 이상)
- [ ] 모바일 메뉴에 로그인/로그아웃 포함
- [ ] 가로 스크롤 없음(body overflow-x: hidden 및 콘텐츠 너비 제한)
